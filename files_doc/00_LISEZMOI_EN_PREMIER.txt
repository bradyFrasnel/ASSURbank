🎓 POUR RÉPONDRE AUX QUESTIONS DU JURY
═════════════════════════════════════════════════════════════════════════════════════

Q: "Pourquoi 4 tables et pas une table Utilisateur générique?"
R: Parce que Banque et Client ont des rôles différents:
   - Banque: gère ses clients (authentification propre)
   - Client: a un compte dans une banque (lié via banque_id)
   - Separation of concerns = meilleure architecture

Q: "Comment vous gérez les virements?"
R: Avec 2 transactions (débit + crédit) dans une même transaction atomique ACID.
   Si une échoue, l'autre échoue aussi. Garantit la cohérence.

Q: "Comment vous protégez les mots de passe?"
R: Avec Argon2 (fonction de hachage sécurisée de Symfony).
   Les mots de passe ne sont jamais stockés en clair.

Q: "Pourquoi Twig et pas React?"
R: Parce que Twig est le moteur de templates natif de Symfony.
   Pour un projet académique, c'est plus cohérent qu'une SPA React.
   Mais le code backend est le même (API REST compatible).

Q: "Les contraintes de sécurité?"
R: CSRF (Cross-Site Request Forgery), XSS, SQL Injection:
   - Symfony Security Bundle gère CSRF automatiquement
   - Twig échappe les outputs par défaut (anti-XSS)
   - Doctrine ORM utilise des prepared statements (anti-SQL Injection)

🚀 PROCHAINES ÉTAPES APRÈS LA SOUTENANCE
═════════════════════════════════════════════════════════════════════════════════════

1. Créer les entités Doctrine
   → symfony console make:entity Banque
   → symfony console make:entity Client
   → etc.

2. Générer les migrations
   → symfony console make:migration
   → symfony console doctrine:migrations:migrate

3. Créer les contrôleurs
   → symfony console make:controller BanqueController
   → symfony console make:controller ClientController
   → etc.

4. Implémenter les services métier
   → Service de virement (VirementService)
   → Service d'authentification
   → etc.

5. Créer les formulaires Symfony
   → make:form VirementFormType
   → make:form ClientFormType

6. Implémenter les templates Twig
   → templates/banque/dashboard.html.twig
   → templates/client/accounts.html.twig
   → etc.
